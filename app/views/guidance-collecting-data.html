Guidance

Data to collect

[This section is very generic. Do we need more focus on open space?]

All evidence base studies require some elements of data and standardised information to be collected. Whilst this will vary from study to study, there are some consistent elements that apply to most elements of a place under study; current population and supply, future supply and future demand, with some analysis required to understand future surplus/deficits.

Similarly we are seeking to collect data consistently, so are requiring core evidence base data to be collected to have:

  • consistent format
  • consistent metadata: metadata summarises basic information about data, making finding and working with data easier
  • typologies: how different factors can be classified, from types of employment land, sports facilities, utilities assets etc
  • associated definitions to explain the typology
  • benchmarks, in some instances can be specified: these are not targets but often averages against which local divergences can be explained and used to guide policy.

How data should be collected

[What data should be collected?]

Data format

Data should be collected in a CSV file, which should include:

  • a header row which must be the first row in the spreadsheet, written exactly as below (headers are case sensitive with no spacing)
  • 1 row for each open space site

We've created an open spaces data CSV template. If it helps, you can use this example file and enter your own data, following the guidelines below.

Column headers:

OrganisationURI

Find your organisation in this list and enter the corresponding Open Data Communities URI.

Typology

Enter the type of open space that the site belongs to.

This should be one of the following:

  • Parks and gardens
  • Natural/Seminatural
  • Outdoor sports facilities
  • Amenity
  • Children and young people
  • Allotments, gardens and farms
  • Churchyards and cemeteries
  • Civic spaces
Reference

Enter the unique reference your organisation uses to identify the open space.

If one doesn’t exist, you need to create one. It should not be used by your organisation to identify any other sites, but can be borrowed from another data set listing the site.

Name

Enter the name your organisation uses for the site.

GeoY

Enter the latitude of a point close to the centre of the site. The value should be 6 or fewer decimal places, using the WGS84 or ETRS89 coordinate systems specified by the open standards for government guidance.

GeoX

Enter the longitude of a point close to the centre of the site. The value should be 6 or fewer decimal places, using the WGS84 or ETRS89 coordinate system specified by the open standards for government guidance.

Be sure you do not mix up the latitude (Geo Y) and longitude (Geo X) values. Any location in the UK will have a latitude (Geo Y) from about 49 to 57 and a longitude (Geo X) from about -7 to 2.
Hectares

Enter the land area of the site in hectares, up to 2 decimal places. Use digits (2) rather than words (two).

OwnershipStatus

Indicate the site’s ownership by entering one of the following values:

  • owned by a public authority
  • not owned by a public authority
  • mixed ownership
FirstAddedDate

Enter the date that the site was first added to this register, in the format YYYY-MM-DD.

LastUpdatedDate

Enter the date this entry in the register was updated, in the format YYYY-MM-DD.

EndDate

If the site no longer needs to be listed (for example, if the site has been built on), it should remain on the register for historical reasons and not be deleted. Enter the date the site was developed or determined to no longer be an open space, in the format YYYY-MM-DD. This field should only be filled in once the site is no longer classified as open space.

Current population and supply

Population

Current population figures, using the ONS population figures for the planning authorities boundary should be used as a baseline. Population should be segmented by age demographics to allow for analysis of availability of more age-specific facilities such as children’s playgrounds.

Open space

Measurement and benchmarking

Hectares (ha) within LPA area of each typology, split between total amount and space which is in public or private ownership.

Benchmarking guidance comes from Fields in Trust’s benchmark guidelines.
